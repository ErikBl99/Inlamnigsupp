Jag siktar på VG

När använder man add respektive commit? - add använder man för att tala om att man har gjort en förändring & att den funkar. Commit använder man för att lägga till färändringarna i sitt lokala repository

Vad är pull repektive push? - pull använder man för att ladda ner innehåll från en remote repository medan push är att skicka nytt innehåll till en remote repository

När använder man log respektive status? - log använder man för att kolla vad som har hänt i en branch. Status använder man för att kolla vad som finns i ens branch, tex vilken branch man står i, om det finns filer att commita, om det finns filer som inte har blivit adderade än

Hur kan man formulera sig i sina commit-meddelanden? - man ska ge en kort förklaring på en rad om ändringar som man har gjort

Extrafråga - Man kan använd git Log eller git Checkout.

Med git log kan man stega sig tillbaka från den senaste commiten eller en i taget.

Git Checkout använder man om vet vilken commit man vill ändra & gå direkt till den.

Varje commit har en speciell tag som man kan använda för att lättare hitta det man letar efter
